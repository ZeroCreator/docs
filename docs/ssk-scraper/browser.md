## 1. Что пробовали: отдельный контейнер с браузером
**Идея была такой:**

Запустить Chromium в одном контейнере, а парсеры — в других. 
Парсеры подключаются к браузеру удалённо через `CDP (Chrome DevTools Protocol)`. 
Это удобно: браузер один на всех, не надо запускать внутри каждого парсера.

**В проекте это действительно реализовали:**

- compose.yml поднимает browser-сервис
- Скрипт browser/docker-entrypoint.sh запускает Chromium с --remote-debugging-port
- Прокси через browser/default.conf перенаправляет порт 9222 → 9223

✅ Гипотеза «можно вынести браузер» была проверена на уровне кода.

## 2. В чём была главная проблема: доступность браузера из парсеров
Как реально работает эта конфигурация
В compose.yml нет network_mode: host для парсеров. Значит:

- Все сервисы (кроме браузера) находятся в дефолтной Docker-сети (или в одной, если явно не указано иное).
- Браузер доступен по имени сервиса browser на порту 9222 (проброшенном на хост, но это не обязательно для внутренней связи).
- Парсеры должны подключаться к браузеру по адресу: `browser:9222` (внутри Docker-сети).

То есть стандартная, правильная схема Docker Compose.

**Гипотеза**
Надо сделать так, чтобы браузер был доступен как 127.0.0.1"

Отсюда родились попытки "починить" доступность через:

network_mode: host (чтобы парсер видел хост-сеть, куда проброшен порт браузера)

ports: "9222:9222" у браузера (чтобы порт был на хосте)

Код, который упорно ходит на 127.0.0.1

Почему эта гипотеза провалилась?
Потому что настоящая проблема была не в доступности:

Docker-сеть работает нормально — если написать host="browser", всё подключается без network_mode: host

**Гипотеза: проблема в том, как парсер ждёт браузер**
→ Тоже нет: парсер падал раньше, чем вообще начинал работать с токенами

📌 Вывод: проблема была именно в канале между парсером и браузером.

watch-trendagent-token-updates использует дефолт 127.0.0.1:9222

Внутри контейнера это не работает (браузер в другом контейнере)

## 4. Пытались починить ретраями, оркестрацией, порядком запуска

5. Что в итоге сделали в проекте
Отказались от внешнего браузера.

В проекте есть коммит (например, bf963eb), где nash_dom (ещё один парсер) переводится с:

```python

nodriver.start(host="127.0.0.1", port=9222)
```

на:

```python
# стало: запускаем браузер локально внутри парсера
run browser locally
```

📌 **Вывод: проект перестал использовать отдельный browser-container как рабочую схему.**
